## What shipped in 1.18.3

Two cross-team-cert messaging fixes:

- **aalk** (continuity-fallback): when the CLI tries to resolve a recipient's address but the registry returns 404, the CLI now falls back to the local known-agents pin store. Closes the case where the sender already has the recipient pinned from prior contact (e.g., reply scenarios).

- **aalm** (authenticated lookup): the CLI now signs registry GET requests using the existing awid signed-read scheme (Authorization: DIDKey + X-AWEB-Timestamp + canonical payload). Closes the case where the recipient's awid row exists with `org_only` or `team_members_only` reachability — anonymous CLI lookups previously 404'd those rows even though the sender was authorized.

Plus 2 server-side fixes for asymmetric branches in messages.py that surfaced when the CLI started populating `to_did` on address-routed mail (previously latent bugs).
